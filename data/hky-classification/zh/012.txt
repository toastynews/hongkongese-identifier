金融 API 完善 銀行B2C才有轉機（石先生）

或許是為了配合特首發展創科的宏願，近來香港金融業擁抱科技的速度明顯有加快。Mobile Payment 雖然還不成氣候，唯爭先恐後推出大燒銀紙，紛紛盼望搶得下一回合的主導權。

與 Mobile Payment 相比，傳統銀行的電子服務仍然是老樣子，用起來不方便，光是查賬功能就很麻煩，一間銀行一個 App 一個登入。在這個人手幾張銀行卡、信用卡的時代，對消費者來說著實困難。

幸好 HKMA 最近擁抱創科的速度也有變快，最近公佈的「銀行業開放應用程式介面（Application Programming Interface, API）框架」就有為這類場景著想。根據文件的說法，第一階段提供查閱銀行服務，第二階段是申請服務，第三階段才是讀取資訊，最後才是進行財務交易。

根據公佈，讀取資訊與進行財務交易還沒有明確的時間表，在沒有明確指引與條例的情況下其實對金融業與消費者都不是一件好事。例如今年初在香港推出的「智能記賬理財 App」– Gini 就是一個好例子。

Gini 利用銀行提供的 API 為用戶提供一站式的銀行賬戶與信用卡查賬服務，只要一個 App 就可以看到所有的消費紀錄，方便簡單，有利消費者謹慎理財。唯在沒有明確框架的情況下，即使其採用不經伺服器儲存資料方式，也利用 256 位元加密技術及匿名化保護，仍然被不少人擔心安全性，確實是一件可悲的事情。
在全球多個國家已經準備開放 API 的時候，香港也必須更快行動，在安全得到保障的前題下，即使用戶不熟悉有關操作也要趕緊推出。否則大家申請那種信用卡較優惠還是得靠「小斯」了。
